# 分享連結改版說明

## 目的與背景
為符合公司「禁止外部連線」的需求，分享連結需具備**可控的 Token 驗證與到期時間**。同時，既有分享連結（無 token）仍須維持原行為，以確保相容性與既有連結可用性。

---

## 實現目標
1. 新增分享連結 **Token 驗證 + 可設定到期時間** 機制  
2. **保留舊分享連結機制**（無 token，仍需登入）  
3. **Token 到期時間由管理員控制台設定**

---

## 行為規則（雙通道）

### ✅ 有 `token` 的分享連結
- 走 **UMC 分享 API**
- **不需登入**
- Token 有到期時間  
- 到期後顯示「連結失效」

### ✅ 沒有 `token` 的分享連結（舊連結）
- 走 **原本分享 API**
- **必須登入**（localStorage token）
- 既有連結不失效，行為完全保留

---

## 管理與設定

### 新增設定
- `SHARE_TOKEN_EXPIRES_IN`  
  - 由管理員設定分享 Token 有效期限  
  - 允許設定 `-1` 代表永不過期  
  - 類似 JWT 的 `expires_in` 格式（可依現有格式維持）

### 管理介面
- Admin Panel 增加「Share Token Expiration」欄位  
- 允許即時更新分享 Token 有效期限  

---

## 相容性說明
- **舊連結不會失效**  
- 新增的 token 分享連結只影響「有 token 的分享連結」  
- 無 token 的分享連結維持原登入驗證流程  

---

## 補充
- `SHARE_TOKEN_EXPIRES_IN = -1` 代表永久有效  
- 管理員可在 Admin Panel 直接調整有效期限  
- 若 token 過期，分享頁顯示失效提示，不跳轉登入
---

## 設定格式範例

`SHARE_TOKEN_EXPIRES_IN` 採用與 JWT 類似的「時間長度」格式（字串）：

- `1d`：1 天  
- `12h`：12 小時  
- `30m`：30 分鐘  
- `-1`：永久有效（不過期）

## API 規格

### 1) 取得分享（新）
**GET** `/api/v1/umc/share/{share_id}?token={share_token}`

- 不需要登入
- Token 過期或無效 → `401`

**Response 200**
```json
{
  "chat": {
    "id": "shared-chat-id",
    "user_id": "shared-<original_id>",
    "title": "Example Chat",
    "chat": { ... },
    "share_id": null,
    "created_at": 1730000000,
    "updated_at": 1730001000
  },
  "user": {
    "id": "user-id",
    "name": "User Name",
    "profile_image_url": "https://..."
  }
}

2) 取得分享（舊）
GET /api/v1/chats/share/{share_id}

必須登入（Bearer Token）
無 token → 401
Response 200

```
{
  "id": "shared-chat-id",
  "user_id": "shared-<original_id>",
  "title": "Example Chat",
  "chat": { ... },
  "share_id": null,
  "created_at": 1730000000,
  "updated_at": 1730001000
}
```

3) 建立分享連結（既有 API，新增 token 回傳）
POST /api/v1/chats/{id}/share

需要登入
回傳內容新增 share_token
Response 200
```
{
  "id": "shared-chat-id",
  "share_id": "shared-chat-id",
  "share_token": "JWT_TOKEN",
  "chat": { ... }
}
```
